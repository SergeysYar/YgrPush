# Обзор системы

## Что это за система

Система прогнозирует итоговое качество варки шампуня до появления лабораторного протокола.

Она использует:

- производственные измерения по этапам варки;
- сведения о партии, продукте и компонентах;
- лабораторные результаты прошлых варок;
- набор ML-моделей, обученных на исторических данных.

Цель системы — заранее подсказать, как с высокой вероятностью закончатся показатели качества партии, чтобы команда производства получила ранний сигнал и могла внимательнее контролировать процесс.

## Какие показатели прогнозируются

Система строит прогноз по трём основным целевым метрикам:

- `pH`
- `viscosity` — вязкость
- `chlorides` — содержание хлоридов

Для каждого показателя возвращаются:

- прогнозируемое значение;
- нижняя и верхняя граница интервала;
- статус относительно норматива;
- уровень уверенности;
- модель, которая дала прогноз;
- количество обучающих варок;
- факторы, повлиявшие на прогноз;
- похожие исторические варки.

## Главная идея

Основная идея системы не в том, чтобы заменить лабораторию, а в том, чтобы дать производству полезный ранний ориентир.

Практически это означает следующее:

- лаборатория остаётся источником итоговой истины;
- система помогает увидеть риск раньше;
- прогноз пересчитывается по мере накопления этапов варки;
- система годится для сопровождения процесса, но не для автоматического управления рецептурой без участия человека.

## Для кого предназначена система

Система полезна сразу нескольким ролям:

- **технологу** — чтобы заранее увидеть риски по качеству;
- **руководителю производства** — чтобы понимать общую ситуацию по варкам;
- **ML/аналитику** — чтобы исследовать влияние этапов и признаков;
- **backend-разработчику** — чтобы встроить прогнозы в другие системы;
- **оператору сопровождения** — чтобы переобучать модели и отслеживать состояние артефактов.

## Что система делает на практике

На практике работа выглядит так:

1. Система читает исторические измерения из `production.db`.
2. Очищает и нормализует значения.
3. Преобразует этапы варки в признаки.
4. Связывает каждую варку с целевыми лабораторными значениями.
5. Обучает модели и хранит их в `ml_storage.db`.
6. По текущей варке строит прогноз на выбранном контрольном этапе.
7. Возвращает результат через API, CLI и dashboard.

## Что важно понимать сразу

- Это **инженерный MVP**, а не финальная промышленная версия.
- Модели уже работают, но надёжность должна подтверждаться на новых варках.
- Часть возможностей реализована как безопасный fallback:
  - optional `CatBoost`
  - optional `SHAP`
- Без некоторых зависимостей система не ломается, а просто понижает уровень доступной функциональности.
