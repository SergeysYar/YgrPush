# Эксплуатация и сопровождение

## Какие файлы важны в эксплуатации

- `production.db` — производственный источник данных
- `ml_storage.db` — ML-реестр, predictions, issues, training runs
- `artifacts/` — бинарные модели
- `reports/` — экспортированные отчёты

## Что хранится в `ml_storage.db`

### `model_registry`

Сведения о моделях:

- id
- target
- type
- status
- version
- metrics
- features

### `training_runs`

История запусков обучения:

- когда обучали;
- какие модели получили;
- какие batch вошли;
- с какими настройками.

### `batch_predictions`

Сохранённые прогнозы по batch и checkpoint.

### `prediction_items`

Сопутствующие признаки, использованные при сохранённом prediction.

### `data_quality_issues`

Сохранённые issues качества данных.

## Рекомендуемый операционный цикл

### После появления новых размеченных варок

1. обновить или проверить `production.db`;
2. запустить summary / validate;
3. переобучить модели;
4. посмотреть `models/summary`;
5. при необходимости промоутить новую модель.

### После накопления predictions

1. запустить `update-actuals`;
2. сравнить факты и прогнозы;
3. оценить качество champion на новых варках.

## Что важно мониторить

- количество размеченных batch;
- качество входных данных;
- число challenger-моделей;
- наличие актуального champion по каждому target;
- число новых predictions без actuals.

## Что делать при проблемах

### Если прогнозы не строятся

Проверить:

- есть ли обученные модели;
- доступна ли `ml_storage.db`;
- есть ли данные по batch;
- не выбран ли пустой checkpoint.

### Если `CatBoost` не участвует

Это может быть нормально:

- библиотека не установлена;
- мало размеченных batch;
- pipeline корректно делает fallback.
