# Ограничения и roadmap

## Ограничения текущего MVP

### 1. Небольшая историческая выборка

Это главный практический ограничитель.

При малом числе размеченных варок:

- сложные модели нестабильны;
- метрики могут быть шумными;
- переносимость на новые данные ограничена.

### 2. Неидеальное качество производственных данных

Система уже умеет это выявлять, но сама по себе она не может исправить первичный источник.

### 3. `CatBoost` и `SHAP` — опциональный контур

Он уже интегрирован, но полностью раскрывается только в корректном окружении и при достаточном числе batch.

### 4. Streamlit — ещё MVP

Интерфейс уже рабочий, но ещё не является финальным промышленным рабочим местом.

### 5. Полный тестовый прогон зависит от окружения

Логика проекта далеко продвинулась, но для полного зелёного тестового контура нужно привести окружение зависимостей в полностью стабильное состояние.

## Что уже считается сильной стороной текущей версии

- корректная batch-aware и snapshot-aware логика;
- разделение production DB и ML storage;
- champion/challenger registry;
- data quality layer;
- сохранение predictions и training runs;
- API, CLI и dashboard уже согласованы между собой.

## Что логично делать дальше

### Приоритет 1

- стабилизировать окружение тестов;
- прогнать полный end-to-end сценарий на реальной базе.

### Приоритет 2

- расширить dashboard:
  - stored predictions
  - stored issues
  - графики training runs

### Приоритет 3

- усилить production polish для `CatBoost + SHAP`;
- формализовать оставшиеся мелкие API-контракты;
- при необходимости добавить более удобную аналитику по drift и качеству.

## Итоговая честная оценка

Текущая система уже является **очень близким к завершению инженерным MVP**:

- ядро работает;
- основные блоки закрыты;
- оставшийся объём — в основном polish, эксплуатация и стабилизация среды.
