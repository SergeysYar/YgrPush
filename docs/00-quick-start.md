# Быстрый старт

Этот документ помогает быстро развернуть систему локально и дойти до первого полезного результата без глубокого погружения во внутренности проекта.

## Для кого этот документ

Подходит для:

- нового разработчика в проекте
- аналитика, которому нужно локально собрать датасет или обучить модель
- инженера, которому нужно быстро поднять API
- пользователя, который хочет понять базовый рабочий сценарий системы

## Что понадобится

- Python `3.12`
- локальная копия репозитория
- файл производственной базы `production.db`
- доступ на запись в каталог проекта для создания `ml_storage.db` и артефактов

## Шаг 1. Установка зависимостей

```bash
python -m pip install --upgrade pip
pip install -e .
```

Если используется виртуальное окружение, сначала активируйте его.

## Шаг 2. Настройка окружения

Создайте `.env` на основе `.env.example` и проверьте ключевые параметры:

```env
DB_PATH=./production.db
ML_STORAGE_PATH=./ml_storage.db
TARGET_PROTOCOL_POLICY=latest
CONFIDENCE_HIGH_MIN_BATCHES=20
```

Минимальная проверка:

- путь `DB_PATH` указывает на рабочую SQLite-базу
- путь `ML_STORAGE_PATH` доступен для записи

## Шаг 3. Проверка данных

Сначала стоит убедиться, что база читается и данные проходят базовую проверку:

```bash
python -m app.cli inspect-db
python -m app.cli validate-data
```

Что это даёт:

- подтверждает доступность базы
- показывает структуру и содержимое ключевых таблиц
- позволяет сразу заметить пустые или проблемные значения

## Шаг 4. Построение и обучение

Для первого рабочего запуска достаточно:

```bash
python -m app.cli build-dataset
python -m app.cli train
```

Результат:

- собирается тренировочный датасет
- обучаются модели по основным таргетам
- артефакты и метаданные сохраняются в `ml_storage.db` и связанных хранилищах

## Шаг 5. Первый прогноз

После обучения можно сделать прогноз для конкретной партии:

```bash
python -m app.cli predict --batch-id 89
python -m app.cli predict --batch-id 89 --up-to-step 4
```

Второй вариант нужен, когда важно получить промежуточный прогноз только по уже завершённым этапам.

## Шаг 6. Запуск API

Чтобы использовать систему из внешнего контура:

```bash
python -m app.cli run-api
```

После этого можно:

- обращаться к API из внешних сервисов
- тестировать маршруты вручную
- использовать dashboard поверх тех же данных и моделей

## Шаг 7. Запуск dashboard

```bash
python -m app.cli run-dashboard
```

Dashboard полезен для:

- анализа партии
- просмотра результатов прогнозов
- объяснения работы системы нетехническому пользователю

## Если что-то пошло не так

- Если не читается база — проверьте `DB_PATH`
- Если не создаются артефакты — проверьте права на запись
- Если модель не обучается — посмотрите качество данных через `validate-data`
- Если неясно, что означает прогноз — откройте `docs/11-faq.md`
